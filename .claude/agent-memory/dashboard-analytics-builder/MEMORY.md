# Dashboard Analytics Builder - Agent Memory

## Project Structure

### Analytics Components Location
- `/components/analytics/` - Reusable chart components
  - `MetricCard.tsx` - Animated metric card with CountUp, trend indicators, sparklines
  - `ResponsiveLineChart.tsx` - Line chart wrapper around Recharts
  - `ResponsivePieChart.tsx` - Pie chart with custom tooltips and labels
  - `ResponsiveBarChart.tsx` - Bar chart component
  - `TimeRangeSelector.tsx` - Date range picker for filtering
  - `ExportButton.tsx` - CSV/PNG export functionality

### Admin Dashboard Architecture
- `/components/AdminDashboard.tsx` - Main admin interface
- `/components/admin/` - New tab-based components (created 2026-02-11)
  - `OverviewTab.tsx` - Dashboard overview with metrics, version info
  - `UsersTab.tsx` - User management with role distribution charts
  - `ContentTab.tsx` - Blog post management and analytics
  - `AISystemsTab.tsx` - AI model configuration and telemetry
  - `SettingsTab.tsx` - System settings, feature flags, audit logs

## Design Patterns

### Chart Library: Recharts
- Used for all charts (Line, Pie, Bar)
- React-native declarative API
- Good TypeScript support
- Responsive by default via ResponsiveContainer

### Animation Library
- `framer-motion` for page transitions and metric cards
- `react-countup` for animated number counters
- Custom CSS animations for scan lines and glows

### Metric Card Pattern
```tsx
<MetricCard
  title="Translated Title"
  value={number}
  previousValue={number}  // For trend calculation
  icon="emoji or ReactNode"
  colorClass="bg-cyan-500/10 text-cyan-400"
  trend="up" | "down" | "neutral"
  onClick={() => {}}  // Optional navigation
/>
```

### Real-Time Data
- Uses `useAppState` hook for Firestore subscriptions
- `onSnapshot` for live updates
- No Redux/Zustand - React Context only

## Performance Optimizations

### Memoization
- `useMemo` for expensive calculations (role counts, filtering)
- `React.memo` for chart components (already applied in analytics/)
- Lazy loading for heavy modals: `React.lazy()`

### Data Aggregation
- Pre-calculate metrics in parent component
- Pass down aggregated data to child tabs
- Avoid re-calculating same data in multiple places

### Virtualization
- Not currently implemented
- Consider for large user/pet lists (>1000 items)
- Use `react-window` if needed

## Common Pitfalls

1. **Hardcoded Strings**: Always use translation keys, never hardcode English text
2. **Non-Responsive Charts**: Always wrap Recharts in `<ResponsiveContainer>`
3. **Missing Empty States**: Every data view needs an empty state component
4. **Firestore Subscriptions**: Remember to unsubscribe in cleanup (`return () => unsub()`)
5. **Type Safety**: Use central `types.ts`, avoid `any`

## Data Transformation Examples

### Role Distribution (Pie Chart)
```tsx
const usersByRole = users.reduce((acc, user) => {
  user.roles?.forEach(role => {
    acc[role] = (acc[role] || 0) + 1;
  });
  return acc;
}, {});

const chartData = Object.entries(usersByRole).map(([name, value]) => ({
  name: capitalize(name),
  value
}));
```
